# Правила рабочего процесса (деплой, Export, Backup)

Этот файл описывает соглашения по работе с кодом и деплою.
Скопируй его содержимое в `CLAUDE.md` нового проекта.

---

## Папка Export (выгрузка на сервер)

После завершения правок **все изменённые файлы копируются в папку `Export/`** в корне проекта с сохранением полной структуры вложенности папок (зеркало production-структуры).

- Пользователь берёт содержимое `Export/` и заливает на FTP — сервер предлагает заменить только изменённые файлы.
- **Claude сам очищает `Export/`** (`rm -rf Export/*`) перед каждой выгрузкой, затем копирует файлы текущей сессии.
- `Export/` всегда должна содержать **только файлы текущей (ещё не залитой) сессии** правок.

Пример: если изменён файл `some/path/file.css`, то в Export создаётся:
```
Export/some/path/file.css
```

## Папка Backup (резервная копия)

Перед внесением изменений в существующие файлы создаётся резервная копия.

- Бекап — это корневая папка `Backup_YYYY-MM-DD/` (или с кратким описанием задачи) с зеркальной структурой путей.
- Содержит оригинальные версии только тех файлов, которые будут изменены в этой сессии.
- Заливается одной операцией на FTP — позволяет откатить изменения.

Пример структуры:
```
Backup_2026-03-18_task-name/
  some/path/file.css
  another/path/script.js
```

## Лог задач (WORKLOG.md)

Файл `WORKLOG.md` в корне проекта содержит историю всех выполненных задач.

Формат записи:
```markdown
## YYYY-MM-DD — Краткое описание задачи

- Что сделано
- Изменённые файлы:
  - `path/to/file1`
  - `path/to/file2`
```

При завершении каждой задачи Claude добавляет новый раздел в `WORKLOG.md`.
